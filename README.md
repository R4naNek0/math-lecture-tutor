# 数学讲义精读助教（math-lecture-tutor）

一个像大学数学老师一样、陪你逐章逐定理精读数学讲义的 skill。它最核心的价值是：**把讲义里被作者一笔带过的推理补全**——凡是原文写"显然""注意到""易知"的地方，它都会一步步讲清楚这些结论到底是怎么来的。

## 它能做什么

1. **按需分块读取**：对厚讲义，自动逐批扫描（每批5页，最多扫到第20页）寻找目录页；找到就直接提取章节结构，找不到则询问你目录位置或整理已扫到的标题——不会一上来就通读几百页。学某章时才精读那章，轻量高效。
2. **定理索引**：首次处理 PDF 时生成 `定理索引.md`，记录全书每个定理/引理/命题的编号、名称和页码。想学某个定理可以直接查索引跳到那页，不用翻全文。
3. **交互式逐定理精讲**：一次只讲一个定理，标注关键条件与结论，严格依据讲义原证明把每一处跳步补全，讲完停下来等你反馈或提问。
4. **维护学习记录**：每讲完一个定理，更新 `学习记录.md`（学到哪、对应页码、你问过什么），下次能直接从页码续学。
5. **复习环节**：每次开始时先问你要不要复习已学内容，复习完再问从哪继续。
6. **沉淀 Typst 笔记**：每个定理在你确认无误后，整理进一份排版正式的 Typst 笔记，写得尽可能详尽，并自动编译检查确保能正常渲染。笔记按章节建文件夹、按小节建文件，方便检索。

## 安装

这是个标准 skill，把 `math-lecture-tutor` 文件夹放进你的 skills 目录即可：

```
<你的 .claude 目录>\skills\math-lecture-tutor\
```

## 怎么用

不用喊它的名字，直接像跟老师说话那样开口就会触发：

- **开始学一份新讲义**：「帮我学这份讲义：`D:\path\to\讲义.pdf`」
- **精讲某个定理**：「讲讲第 3 章那个收敛定理的证明」
- **续学**：「继续学」——它会读学习记录，先问你要不要复习，再问从哪接着学

学习过程中你可以随时打断提问、要求把某一步再展开。觉得一个定理讲明白了，告诉它「懂了」，它就会更新记录、整理 Typst 笔记，然后进入下一个。

## 产出文件

都放在讲义 PDF 所在的同一文件夹下：

| 文件/文件夹 | 内容 |
|---|---|
| `讲义大纲.md` | 章级轻量大纲（章节名+页码范围）；学某章时追加该章详细大纲 |
| `定理索引.md` | 全书定理/引理/命题的编号、名称、页码，快速定位用 |
| `学习记录.md` | 学习进度（含页码）+ 你提过的问题 |
| `Typst笔记/<第X章-章节名>/` | 每章一个子文件夹，下按小节建 `X.Y-小节名.typ` |

## 依赖：Typst（用于笔记渲染检查）

笔记沉淀这一步会调用 `typst compile` 做语法检查，确保交付的 `.typ` 文档能正常渲染。所以需要本机装有 [Typst](https://github.com/typst/typst)，并且 `typst` 命令在 PATH 里（在终端敲 `typst --version` 能出版本号即可）。

- 没装 Typst：精讲、大纲、学习记录、复习等功能都正常，只是笔记的"渲染检查"会跳过或失败——`.typ` 源文件仍会生成，你可以自己择时编译。
- 笔记模板内置在 `assets/template.typ`，用了 `ctheorems` 包做定理框，首次编译时 Typst 会自动下载该包。
- 中文显示需要本机有中文字体（模板默认尝试 `Source Han Serif SC` 等）。字体缺失只是警告，不影响文档生成。

## 自定义笔记样式

想改笔记的外观（配色、字体、封面、定理框样式），直接编辑 `assets/template.typ`——所有新笔记都基于它生成。
